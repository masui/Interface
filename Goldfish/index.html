GoldFish - JavaScriptベースの実世界GUIフレームワーク

2月号の記事で、 パソコン上のGUIのような操作を実世界で行なうことができるようにする「実世界GUI」を紹介しました。 実世界GUIを実現するには様々な手法が考えられますが、 最近のAndroidに内蔵されているNFCリーダと ブラウザ上のJavaScriptを活用することによって実世界GUIを実現する 「GoldFish」というフレームワーについて紹介します。 SuicaやTaspoのような手軽なRFIDタグと NFCリーダ内蔵Androidを利用することにより、 安価に手軽に実世界GUIを実現することが可能です。

GoldFishを利用すると、 Androidに内蔵されたNFCリーダと動きセンサを利用して 様々な実世界GUIを簡単に実現することが可能になります。 GoldFishクライアントをインストールしたAndroidと Webページ上のJavaScriptプログラムを用意しておくことにより、 「音量調整」と記述したNFCタグにAndroidを近付けてから 回転させることによりテレビの音量を調整したり、 NFCタグを貼ったパソコン間でデータをコピー/ペースト(コピペ)したり、 NFCタグを貼ったドアの上でAndroidを操作してドアを開けたり、 といったGUI的操作をあらゆる場所で実現することができます。

下の写真はGoldFishを使ってパソコン間でデータをコピペしているところです。 金魚すくいのようにAndroidをひねってパソコンのデータをすくい上げた後、 別のパソコンにデータを流し込んでいます。


実世界コピペ

GoldFishのアーキテクチャ

GoldFishでは、 Android上で動作する GoldFishクライアントと GoldFishサーバを以下のように組み合わせて 実世界GUIを実現します。
  1. NFCタグのIDとWebページのURLの対応をGoldFishサーバ(http://ubif.org/)に登録しておく
  2. AndroidでNFCタグにタッチすると、タグに対して登録してあるWebページが自動的に開く
  3. ページ内容がAndroidに表示され、センサ情報を扱うJavaScriptプログラムが動く
Androidのインテント機能、通信機能、NFCリーダ、動きセンサを利用することにより このような連携が可能になっています。

Androidのインテント

Androidには プロセス間通信を行なうための「インテント」という便利な機能があり、 GoldFishでは AndroidをNFCタグにタッチしたとき自動的にWebページが開くようにするため インテント機能を利用しています。

Androidのインテントとは プロセス間でイベント的にデータを受け渡しするための仕組みで、 異なるアプリケーション間でデータを受け渡したり アプリケーションを起動したりするのに利用します。 たとえば、撮影した写真を加工してメールで送りたいとき、 パソコンを利用する場合は まずデジカメで撮った写真ファイルを画像編集アプリケーションで読み出してから編集し、 編集後のファイルをセーブしてから メーラの添付ファイルとして指定するといった面倒な手順が必要ですが、 Androidの場合は 写真撮影アプリケーションから画像編集アプリケーションをインテントとして呼び出し、 同様にして画像編集アプリケーションからメールアプリケーションを呼び出すことができるので、 画像ファイルをセーブしたりコピペしたりすることなく、 写真撮影⇒加工⇒送信、という作業を連続して行なうことができるようになっています。

上の例では画像データがインテントとして利用されていますが、 Androidでは様々な形式のデータをインテントとして利用することができます。 NFC読み取りもインテントとして定義されています。 AndroidをNFCタグに近付けると タグ情報が自動的に読み取られてNFCインテントが生成されるので、 GoldFishのクライアントアプリケーションを デフォルトアプリケーションとしてNFCインテントに対応づけておくことにより、 自動的にGoldFishクライアントを起動することができます。 つまりAndroidをNFCタグに近付けるだけでGoldFishアプリケーションが起動することになります。 認証にQRコードを使う場合は QRコードを読み取るアプリケーションを立ち上げてから AneroidをQRコードに近付ける必要がありますが、 NFCのインテントを利用する場合はアプリケーションが自動的に立ち上がるので便利です。

Androidのアプリケーションは自分がどういうインテントを受け取ることができるかを定義しています。 画像を利用できるアプリケーションは画像インテントを受け取り可能であると定義してあり、 画像を出力するアプリケーションからそのようなアプリケーションを選ぶことができます。 メールやtwitterなどのアプリケーションは画像インテントを受け取れるようになっているので、 カメラで写真を撮影した後で 下図のようにこれらのアプリケーションを選択することができます。

NFCインテントを処理可能なアプリケーションが複数ある場合、 AndroidケータイをNFCにタッチした後で、 下図のようにアプリケーションを選択することができます。 ここでは、 NFCを利用できるアプリケーションが2個あるので 二つの選択枝が表示されています。

NFCインテントで起動するアプリケーションを複数利用するのは大変なので、 デフォルトのNFC読み取りアプリケーションを用意しておいて あらゆるNFCタグに対してそのアプリケーションが自動的に起動するように設定し、 NFCデータに応じて異なる動きをさせるのがよいと思われます。 GoldFishでは以下のような手法をとることにより、 Androidにインストールするアプリケーションの数を増やすことなく 各種のNFCに対応ができるようにしています。

  • NFCインテントにより常にGoldFishアプリケーションが自動的に起動される
  • GoldFishアプリケーションは Androidのセンサが利用できるように初期化したブラウザ要素を表示し、 NFCのIDに対応したWebページを表示する
たとえばNFCのIDが12345のとき、GoldFishは http://ubif.org/12345 というURLにアクセスします。 ubif.org上では 12345のようなIDと別URLの関係を登録しておき、 登録されているURLにリダイレクトします。 たとえば、 ubif.org上で 「12345 ⇔ http://example.com/」 という対応が定義されていた場合、 12345というIDをもつNFCを読み取ったとき example.comが表示され、 ページ内でJavaScriptプログラムが存在する場合にはそれが実行されます。

NFCのIDをもとに実際のURLへの変換テーブルを利用することにより、 任意のWebページを利用することができますし、 アプリケーションをブラウザ上のJavaScriptで動かすという方法を 利用しているため、 NFCに対応したアプリケーションはWeb上のJavascriptで記述しておけばよく、 Androidにアプリケーションをインストールする必要がありません。

Androidのアプリケーションは通常はJavaで記述されますが、 Javaはプログラム言語自体が複雑ですし、 センサの値の監視/画面の更新/通信の実行/を同時に行なうような 並行処理プログラムを簡単に記述することができません。 JavaScriptはJavaよりもシンプルであり、 関数をファーストクラスオブジェクトとして扱えるため タイマやとコールバックを用いた並行処理の記述が容易です。

GoldFishは、 NFCを読み取ってブラウザを呼び出すAndroidアプリケーション (GoldFishクライアント)と、 GoldFishブラウザがアクセスするサーバ (GoldFishサーバ) というふたつの要素で実現されています。

GoldFishクライアント

GoldFishクライアントは NFCインテントを受け取ってIDを取得した後、 センサを利用できるようにしたWebViewのセットアップを行ない、 WebViewからhttp://ubif.org/(ID)というアドレスにアクセスするプログラムです。 GoldFishクライアントはAndroid Marketで公開されており、 ソースは http://github.com/shokai/GoldFish で公開しています。

GoldFishはJavaで実装したアプリケーションで、 WebViewコンポーネントでwebページを表示します。 WebView内のJavaScriptとAndroidネイティブのJavaが通信する事によって、 JavaScriptからセンサーなどの機能を呼び出すことができます GoldFishはNFCタグを読んだ際に、あらかじめタグに対して登録されている URLを読み込み、WebViewに表示する。NFCタグはGoldFishのwebサイトで登録できます。

GoldFishサーバ

GoldNFCは WebViewブラウザを開いた後、 ubif.org/IDにアクセスし、 そのIDに定義されている外部ページに移動します。 bit.lyや*.coのように、 長いURLを短くしてくれる短縮IDサービスが近年よく利用されていますが、 これと同じ方法を利用することによって ubif.org/zzzzzからXXXXXXのようなURLへの移動を実現しています。 ubif.orgではこれと同じ仕組みを利用し、 ubif.org/xxxxのようなURLから XXXXXXのようなURLにジャンプさせることができます。 たとえば、Androidのセンサを利用するJavaScriptプログラムを http://example.com/android というページに記述しておき、 NFCのxxxxというIDをhttp://example.com/androidに対応させておけば、 AndroidでNFCにタッチしたとき ubif.org経由で ブラウザはhttp://example.com/androidのページを読み込み、 そこに記述されているHTMLやJavaScriptがロードされてAndroidのブラウザ上で表示/実行されることになります。

..... GoldNFCはUBIF.orgのhttp://ubif.org/(ID) にアクセスするので、 定義に従ってそれをリダイレクトします。 IDを含むURLへのアクセスがあったとき、 moved permanentlyというステータスを返すことにより これを実現しています。

GoldFishを利用した実世界GUIシステムの実例

GoldNFCを搭載したAndroidをNFCタグに近付けると ブラウザが起動して http://UBIF.org/(NFCのID) にアクセスします。 http://UBIF.org/(NFCのID) へのアクセスは、ubif.orgにおいてそのIDに登録されている URLにリダイレクトされます。 たとえば UBIF.orgにおいて ID=xxxx => http://example.com/ という対応を登録しておけば、 xxxxというIDをもつNFCタグにタッチすると example.comの内容が 標示されます。 そのページにJavaScriptプログラムが記述されている場合は それが実行されます。

関連情報やマニュアルの表示

パソコンの設定や機器の使い方がわからなくことがよくありますが、 パソコンや機器の上にNFCタグを貼っておき、 そのタグのIDをubif.org上でWikiやマニュアルページのWebに対応づけておけば、 パソコンや機器にAndroidを近付けるだけで機器の設定状態やマニュアルを ブラウザで見ることができます。 この場合は特別なプログラミングは必要ありません。

回転GUI

上の例ではタッチした後で操作は行ないませんでしたが、 AndroidをNFCにタッチした後で回転することによって 音量ツマミのようなGUIを実現する例を示します。 以下のようなHTMLとJavaScriptをもつWebページを用意し、ubif.org上で NFCタグのIDとの対応を定義しておきます。 goldfish.jsはセンサを利用するための初期設定をするスクリプトで、 これによって gyro = goldfish.gyroscope(); のようにしてジャイロのようなセンサを利用できるようになります。 http://pitecan.com/GF/clock.html 
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
    <title>ダイヤルGUI</title>
    <style type="text/css">
    body {
      backgound-color: #eee;
    }
    </style>
    <script src='goldfish.js' type='application/javascript'></script>
  </head>
  <body>
    <canvas id="canvas" style="width:320px;height:400px;"></canvas>
    <script type="text/javascript">
    var radius = 80;
    function drawCircle(x,y,radius){
      canvas = document.getElementById('canvas');
      canvas.width = x * 2;
      canvas.height = y * 2;
      ctx = canvas.getContext('2d');
      ctx.beginPath();
      ctx.fillStyle = 'rgb(40,40,40)';
      ctx.arc(x, y, radius, 0, Math.PI*2, true);
      ctx.fill();
    }
    setInterval(
      function(){
        gyro = goldfish.gyroscope();
        radius -= (gyro.z * 2);
        if(radius > 160) radius = 160;
        if(radius < 1) radius = 1;
        drawCircle(160,200,radius);
      }, 50
    );
    </script>
  </body>
</html>
このプログラムではsetInterval()によって 50msごとにジャイロをチェックするスクリプトが動くようになっており、 ケータイの水平回転角を取得することができます。 その値によって直径を変化させた円を描くことにより、 回転に従って円の大きさが変わるGUIを実現できます。

実世界コピペ

冒頭で紹介した「実世界コピペ」は コンピュータ間でAndroidを用いてコピーアンドペーストができる アプリケーションです。 あらかじめコンピュータにNFCタグを貼り付けておき、 そこにAndroid端末で触れると金魚すくいの画面が表示されます。

実世界コピペ
Android端末をパソコンにタッチしてから右にひねると 現在最前面に表示しているwebページがAndroidにコピーされ、 Android端末を別のコンピュータにタッチしてから 左にひねるとペーストが行われます。

ここでは、GoldFishアプリケーションが 各コンピュータと直接通信を行なうのではなく、 GoldFishアプリケーションおよび各コンピュータが インターネット上のクリップボードサーバと通信しています。 各パソコンには クリップボードクライアントがインストールされており、 パソコンに貼り付けられたNFCタグのIDが実行時に与えられています。

Androidクライアント上で動作するJavaScriptプログラムは goldfish.accelerometer()関数を用いて10msec毎に加速度センサを監視します。 1G以上の加速度が100msec以上連続で左右どちらかにかかったとき、 コピペジェスチャだと判断しています。 ジェスチャが判定されると、 goldfish.id()関数、goldfish.tag()関数で 端末IDとNFCタグのIDを取得し、 ジェスチャ入力値/端末ID/NFCタグIDを クリップボードサーバに送信します。 各コンピュータにインストールされたクリップボードクライアントは、 Cometを用いてクリップボードサーバから通信があるまで待機しています。 URLが送られてきた時はWebページを開き、 コピー命令が送られてきた時は現在開いているURLを返信します。 このように、Andoidとパソコンを協調して動作させることにより 実世界コピペを実現することができます。

パソコンやプリンタの間でデータをコピーする場合、 機器同士が隣にある場合でも データ送信先の名前を入力したりアイコンをクリックしたりする必要がありましたが、 実世界コピペを使えば直接指示でデータ送信できるようになります。

GoldFishの可能性

GoldFishを使うと様々な実世界GUIアプリケーションを簡単に実装することができます。 今回は簡単な例を少しだけ紹介しましたが、 「AndroidとGoldFishで何ができるのか?」という問いは、 「パソコンとマウスで何ができるのか?」という問いと同じで、 ほぼ何でもできるというのが答になるでしょう。 GoldFishを利用すると、 通常のWebページで利用できるインタラクションに加え、 操作対象の指定、ジェスチャーやGUIの操作、 ユーザとコンテキストの判別、 他のシステムとの通信などを JavaScriptプログラムとして簡潔に記述できます。 GoldFishを利用するためには このようなJavaScriptプログラムをWeb上のどこかに用意しておき、 ubif.orgにおいてそのURLをタグのIDと関連付けるだけであり、 各端末にプログラムをインストールする必要がありません。

GoldFish によって実世界GUI開発のハードルが大きく下がったといえるでしょう。 様々な場面で活用していただければと思います。
